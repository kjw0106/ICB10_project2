# 네이버 오픈API 공통 가이드 (요약)

네이버 오픈API를 사용하여 애플리케이션을 개발하기 전에 알아야 할 기본 공통 사항을 기술합니다.

## 1. 오픈API 개요
네이버 오픈API는 네이버 플랫폼의 다양한 기능을 외부 개발자가 편리하게 사용할 수 있도록 웹 API 또는 SDK 형태로 제공하는 서비스입니다. 네이버 로그인, 검색, Clova AI(음성 인식/합성, 얼굴 인식), Papago 기계 번역 등이 있습니다.

## 2. API 로그인 방식에 따른 분류
* **로그인 방식 오픈 API**
  * 네이버 아이디 로그인 연동 후 획득한 사용자 Access Token을 사용하여 API를 호출합니다.
  * 개인정보 보호 및 사용자 권한이 필요한 API에 사용됩니다.
* **비로그인 방식 오픈 API**
  * 별도의 로그인 없이 HTTP 요청 헤더에 발급받은 **Client ID**와 **Client Secret**을 포함하여 호출할 수 있습니다.
  * 검색, 데이터랩, 파파고 번역 등의 서비스가 이에 해당합니다.

## 3. 사전 준비 사항 (애플리케이션 등록)
API를 호출하기 위해서는 먼저 네이버 개발자 센터에서 애플리케이션을 등록하고 인증 정보를 발급받아야 합니다.
1. **네이버 개발자 센터 접속**: [Application > 애플리케이션 등록] 선택
2. **이용약관 및 계정 정보 등록**: 최초 1회 휴대폰 본인 인증 필요
3. **애플리케이션 상세 설정**:
   * **애플리케이션 이름**: 최대 40자 설정
   * **사용 API**: 연동할 API 서비스 추가 (예: 검색, 데이터랩 등)
   * **서비스 환경**: 웹(URL 입력), 모바일 앱(패키지명 입력) 등 연동 대상 환경 설정
4. **인증 키 발급**: 등록이 완료되면 **Client ID**와 **Client Secret** 확인 가능
5. **주의 사항**: 키 노출 및 보안을 위해 단체 회원 가입을 통한 관리가 권장됩니다.

## 4. 공통 오류 코드 개요
* API 호출 시 인증 실패(401), 권한 없음(403), 잘못된 요청(400), 시스템 오류(500) 등 다양한 상태 코드가 반환됩니다.
* **403 Forbidden** 에러 발생 시, 등록한 애플리케이션 설정의 **API 설정** 탭에서 해당 API 사용이 활성화되어 있는지 가장 먼저 체크해야 합니다.
